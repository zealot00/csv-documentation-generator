## 风险评估填充规则

### ID编号规则

| 前缀 | 类别 |
|------|------|
| DI | 数据完整性 / Data Integrity |
| AV | 系统可用性 / Availability |
| SC | 安全性 / Security |
| CP | 合规性 / Compliance |
| PF | 性能 / Performance |
| OP | 运营 / Operational |

### 严重性评分规则

| 评分 | 等级 | 判断标准 |
|------|------|---------|
| 5 | 严重 | 影响患者安全或导致合规问题 |
| 4 | 高 | 主要功能丧失、业务中断 |
| 3 | 中 | 部分功能受影响 |
| 2 | 低 | 次要功能受影响 |
| 1 | 可忽略 | 几乎无影响 |

### 层级风险规则

> **GAMP 5 M12批判性思维核心规则**:
> - 父功能(如GxP数据操作)确定基础风险等级
> - 子功能(如数据录入)风险不得高于父功能
> - 如果子功能看起来更高，重新评估父功能

### RPN计算

```
RPN = 严重性(S) × 可能性(L) × 可检测性(D)

RPN 1-25: 低风险 - 可接受
RPN 26-50: 中风险 - 需要缓解措施
RPN 51-75: 高风险 - 必须采取缓解措施
RPN 76-125: 严重风险 - 暂停上线
